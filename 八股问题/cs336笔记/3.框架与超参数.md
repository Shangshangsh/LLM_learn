# 模型框架

这一个部分我们需要学习一下现在主流的大模型框架都是由哪些部分组成的



但是在这之前，我们需要先大概了解一下数据的stream都经过了哪些

## Residual stream

<img src="https://cdn.jsdelivr.net/gh/ShiangHou/markdown_image/img/image-20260504002735642.png" alt="image-20260504002735642" style="zoom:80%;" />

首先当然是转成token，此时我们的维度是batch，sequence

然后我们进入embedding层，这一个embedding主要是“查表”，每一个词嵌入维度后就是

B，S，d

然后进入我们的Residual Stream

我们会穿过N个decoder层，

对于每一个decoder层，会有一个MHA以及FFN

MHA其实是Causal Multi-Head Self-Attention

**Pre-Norm (前置归一化)**：输入 $X_l$ 首先经过归一化层（现代模型多采用 RMSNorm）。

- $\tilde{X}_l = \text{RMSNorm}(X_l)$，形状保持 $[B, L, d]$。

**线性投影 (Q, K, V Projections)**：将 $\tilde{X}_l$ 乘以权重矩阵 $W_q, W_k, W_v$，映射到查询、键和值空间，并根据注意力头数 $h$ 进行切分。

- 若采用 RoPE（旋转位置编码），则在此处对 $Q$ 和 $K$ 张量应用复数平面上的旋转变换，注入相对位置信息。

**计算注意力分数 (Scaled Dot-Product Attention)**：

- 计算 $Q$ 和 $K^T$ 的内积。
- **应用因果掩码 (Causal Mask)**：将下三角矩阵之外（即未来词元对当前词元的注意力）的值赋为 $-\infty$，确保在第 $t$ 步时，模型只能看到 $\leq t$ 的上下文。
- 应用 Softmax 获取注意力权重，再将其与 $V$ 相乘获取当前头的上下文表征。
- $\text{Attention} = \text{Softmax}(\frac{Q K^T}{\sqrt{d_k}} + M)V$

**输出投影与残差连接**：拼接所有头的输出，进行一次线性映射（$W_o$），然后加回主干残差流。

- $H_l = X_l + \text{Attention\_Output}$

然后就是下一步，进入到一个FFN层

**Pre-Norm**：同样先对残差流状态进行归一化。

- $\tilde{H}_l = \text{RMSNorm}(H_l)$

**MLP/FFN (升维与降维)**：进行多层感知机映射。现代架构普遍采用基于门控线性单元（如 SwiGLU）的变体。

- 通常包含两个并行的向上投影（Up-projection 和 Gate-projection），映射到更高维度（如 $\frac{8}{3}d$ 或 $4d$），执行元素级乘法和非线性激活，最后通过向下投影（Down-projection）映射回维度 $d$。

**残差连接**：

- $X_{l+1} = H_l + \text{FFN}(\tilde{H}_l)$



最后经过N层相同的decoder后，我们抵达最后，

**Final Norm**：对最后输出的 $X_N$ 应用最后一次归一化。

- $X_{out} = \text{RMSNorm}(X_N)$

**LM Head (Unembedding Layer)**：执行一次线性映射，将隐藏层维度 $d$ 映射回完整的词表大小维度 $|V|$。

- **张量形状变化**：$[B, L, d] \times [d, |V|] \rightarrow [B, L, |V|]$。

这个张量被称为 **Logits**，即模型对词表中每一个 token 在每一个时间步的未归一化预测得分。



在得到logits后，我们下一步就是看是输出还是训练

**训练阶段 (Training)**：

- 对 Logits 的最后一个维度应用 Softmax 函数，将其转化为概率分布 $P$。
- 提取真实标签（即输入序列向右平移一位的词元，Next Token），计算交叉熵损失（Cross-Entropy Loss），并通过反向传播更新整个网络的权重矩阵。

**推理阶段 (Inference/Generation)**：

- 在时刻 $t$，我们只关注 Logits 张量在序列维度上的最后一个切片 $[B, 1, |V|]$，这代表对第 $t+1$ 个 token 的预测。
- 应用 Softmax 转化为概率分布。
- 通过特定的解码策略（如 Greedy Search 贪心获取最大概率索引，或 Top-p/Top-k 截断采样，或带温度系数 Temperature 的多项式采样）从该分布中选取出一个具体的 Token ID。
- **自回归循环**：将这个新生成的 Token ID 追加到原始输入序列的末尾，作为一个新的输入，重新进入步骤 1（在实际工程实现中，这一步会引入 KV Cache 机制来避免重复计算前面的 token，从而将矩阵乘法降级为矩阵向量乘法，优化算术强度）。

以上就是一个大模型的框架

![image-20260504105510955](https://cdn.jsdelivr.net/gh/ShiangHou/markdown_image/img/image-20260504105510955.png)



其实很多之前我们学习的都是什么勾把 transformer，这种又是encoder又是decoder和现在大模型主流的框架相差很远，而且现在模型框架有很多改动的地方，本次我们就来看一下一些关键的组件是怎么改动的，我们主要看几点

1：归一化的改进，我们有pre-norm、RMSnorm，去掉bias

2：激活函数，GELU，gate，等

3：位置层编码，这个现在通用的就是ROPE

4：网络层结构（串行/并行），这个主要是和显卡训练有关

5：超参数的设计。我们提到，比如FFN升维和降维这个比率，多头注意力头的维度、框架的宽度和深度的比率，词表大小，weight decay（dropout这样正则）

6：训练稳定性：输出softmax的稳定性，注意力softmax的稳定

7：注意力机制的优化：MQA、GQA、SWA

## Architecture

### 归一化

#### pre-norm /post-norm

在标准的transformer中，我们使用的是post-norm

回到我们刚才residual stream中去看，

<img src="https://cdn.jsdelivr.net/gh/ShiangHou/markdown_image/img/image-20260504105926316.png" alt="image-20260504105926316" style="zoom:80%;" />

就是我们的输入x，经过一层MHA，然后残差相加，然后再进行norm的归一化，然后往下传

也就是说，x+attentoion_x中，这个x是归一化的，然后整体归一化作为下一个x往下传

仔细观察可以发现，这相当于是 把residual stream中的x进行了归一化！因为每次传的时候，x总是经过了归一化的



而pre-norm的做法是，这个x在进入MHA前进行归一化，然后attention后和x相加，作为下一个输入

此处x+attention_x中，attention_x是”运算前归一化“，即使是输出的attention_x也没有归一化，最后二者合并，在进入下一层前在归一化

也就是说，这个归一化过程并没有在residual stream中，



那么原因是什么，

1：首先将norm从residual stream移除，自然就是说明这个norm会影响残差流的传递，即pre norm会让残差流保持畅通

2：Gradient Attenuation，即梯度衰减，有研究表明（看下图），随着网络层的增加，post norm会导致权重的衰减，而pre norm不会

![image-20260504111125187](https://cdn.jsdelivr.net/gh/ShiangHou/markdown_image/img/image-20260504111125187.png)

3：Gradient spikes 梯度尖峰，意识就是说Pre-norm 在训练过程中全局梯度范数（Global norm）的波动更小，更加平稳

4：**取消 Warm-up 与使用更大学习率**：最初采用 Pre-norm 的直接好处是可以移除学习率的 warm-up 阶段，而在今天，它最大的优势是能为大型网络提供更好的稳定性，并允许使用更大的学习率进行训练



这里需要去解释一下

首先是warm-up是什么意思

先说操作：再训练开始的几个step或者epoch的时候，不适用我们设置好的学习率，而是让学习率从0开始慢慢到我们设置的学习率



**为什么需要warmup**

在post norm的时候，我们一般都需要，

看一下post norm

$x_{l+1} = \text{LayerNorm}(x_l + \text{Sublayer}(x_l))$

当模型刚初始化时，所有参数都是随机的。在这个结构中，误差在反向传播时，必须一次又一次地穿过 `LayerNorm` 层。

研究表明（讲义中引用的 Xiong 2020 论文），这种结构会导致**靠近输入端的底层（浅层）梯度极小，而靠近输出端的顶层（深层）梯度极大**。  

同时，在这个阶段极容易出现**梯度尖峰（Gradient Spikes）**。

直接给它一个很大的学习率，巨大的梯度乘以巨大的学习率，参数瞬间就会更新到一个极其离谱的值，模型直接“死掉”（Loss 变成 NaN）。因此，在 Post-norm 时代，**必须**依赖长期的 Warm-up，小心翼翼地让参数慢慢走到一个相对安全的区域，才敢发力。



当我们用pre norm的时候，就是

$x_{l+1} = x_l + \text{Sublayer}(\text{LayerNorm}(x_l))$

观察可以发现，误差并没有走残差，直接传递过去了，这样的话，在冷启动阶段，第 100 层（深层）和第 1 层（浅层）接收到的梯度大小也是基本一致的，不会出现深层撑死、浅层饿死的情况（Gradient Expectation 图表证明了这一点）



这样的话 ，好处是

**极高的稳定性**：因为每一层的梯度大小都很均匀且平缓，没有了“梯度尖峰”，模型在训练初期的步伐非常稳健。  

**敢于使用更大学习率**：既然一开始路就很平，车子就不会失控。这就意味着我们可以大幅度缩短甚至取消 Warm-up 阶段，并且**敢于把最大学习率（峰值学习率）调得更高**

#### laryer norm/rms norm

刚才说的是在哪里进行归一化，现在看一下怎么进行归一化

#### layernorm

这个就是做了一个标准化

$y = \frac{x - E[x]}{\sqrt{Var[x] + \epsilon}} * \gamma + \beta$

很好理解，减去均值，除方差，此外这个层还会乘一个系数（缩放）并加上一个bias（平移）用来学习

看一下这个操作，需要算一个均值，算一个方差，然后一个减法，一个除法，一个根号，

#### rmsnorm

$y = \frac{x}{\sqrt{||x||_2^2 + \epsilon}} * \gamma$

这个就是直接除以自身的均方根（RMS），

我是笨b，什么是均方根

md就是每一个元素平方，然后加一起，/n，然后再开根号

$$x_{rms} = \sqrt{\frac{x_1^2 + x_2^2 + \dots + x_n^2}{n}}$$

感觉讲义上是不是写错了，还少了一个除n，也就是除d

这个的好处是，可以减少计算量，

我是笨比想不出下面的问题：



你可能会问：计算一个均值、加一个偏置项，对 GPU 来说连九牛一毛的计算量都算不上，为什么要费劲去砍掉它？

这就引出了讲义中极其重要的一页：**理论计算量（FLOPs）不等于实际运行时间（Runtime）**。  

讲义给出了一个令人震惊的数据对比：

- **矩阵乘法（Tensor contraction）**：占了整个模型 **99.80%** 的理论计算量（FLOPs），但只占了 **61.0%** 的实际运行时间。  
- **统计归一化（Stat. normalization，即 LayerNorm）**：仅仅占了 **0.17%** 的计算量，却占了高达 **25.5%** 的实际运行时间！  

**为什么会这样？答案是：数据移动（Data Movement / 访存限制）**。  

- GPU 的计算核心（ALU）非常快，但显存读写速度相对较慢。
- 矩阵乘法是“计算密集型”的（FLOP-to-memory 比例高达 153），GPU 可以一次性读入一点数据，然后疯狂计算很久，计算单元没有闲着。  
- LayerNorm 是极度的“访存密集型”（FLOP-to-memory 比例极低，只有 3.5）。GPU 需要为了算一个均值遍历读取一遍数据，为了算方差再读取一遍，最后相加减还要读写。在这个过程中，GPU 的计算核心在“干等”显存把数据搬过来。  

**RMSNorm 的胜利就在于此**：它去掉了均值计算，极大减少了对显存的重复读取；它去掉了偏置项 $\beta$，意味着每次计算时不需要再从显存里把 $\beta$ 这个张量搬运到计算核心里。虽然省下的计算量微乎其微，但省下的“数据搬运时间”（Wallclock time）是非常可观的。  

原来如此，所以其实是为了减少训练的时候消耗的时间，极大的减少了数据读取转移的时间

#### Bias

继续延申，

既然明白了“搬运 Bias 张量极其拖累显存带宽”这个道理，现代大模型干脆一不做二不休，把网络里其他地方的 Bias 也全部拔掉了

$FFN(x) = \max(0, xW_1 + b_1)W_2 + b_2$

现在直接斩杀bias

$FFN(x) = \sigma(xW_1)W_2$

原理也很简单，就是

**节省内存带宽**：少存储、少搬运 Bias 张量，提升硬件执行效率。  

**优化稳定性**：在某些情况下，去掉这些常数项反而能带来更好的优化稳定性。

#### 门控GLU

**什么是“门控（Gating）”？** 传统的 FFN 是串行的（线性 -> 激活 -> 线性）。而 GLU 引入了一个**并行的分支**。它不再仅仅是对 $xW_1$ 做激活，而是同时计算另一个独立的线性映射 $xV$，然后将这两者的结果进行**逐元素相乘（$\otimes$）**

也就是说，门控又搞了一个参数矩阵V，把激活后的东西乘以这个门控

例如，如果我在ReLU加了门控，那么就是$\max(0, xW_1) \otimes (xV)$

注意这里是逐元素相乘

然后整体的FFN就是$FF_{ReGLU}(x) = (\max(0, xW_1) \otimes xV)W_2$

这个操作本质上是让模型自己动态地学习“应该让哪些特征通过，哪些特征被抑制”，使得表达能力大幅增强

#### SwiGLU 和 GeGLU

GeGLU，我们照猫画虎，直接写就是

$FF_{GeGLU}(x, W, V, W_2) = (GELU(xW) \otimes xV)W_2$

SwiGLU就是

$FF_{SwiGLU}(x, W, V, W_2) = (Swish_1(xW) \otimes xV)W_2$

我是笨比，这个Swi是什么

$\text{Swish}(x) = x \cdot \text{sigmoid}(x)$

其实就是Sigmoid Linear Unit

sigmoid知道吧，$\frac{1}{1+e^{-x}}$

这个和那个高斯其实差不多，x很大的时候，sigmod就是1，x是0就是0.5，很小的时候就是0

和RELU相比，这个和GELU的优势差不多，但是

GELU 依赖于复杂的误差函数或基于 $\tanh$ 的复杂近似公式。相比之下，Swish 只需要算一个简单的 Sigmoid 再乘以自身。在很多现代 GPU 算子库中，计算 $x \cdot \text{sigmoid}(x)$ 已经被优化到了极致

就很舒服



你可能注意到了，GLU 变体多了一个并行的权重矩阵 $V$。如果保持原来的隐藏层维度（$d_{ff}$）不变，模型的参数量和计算量都会大幅增加。  

为了保证公平（即在相同的总参数量下进行对比），**使用 GLU 变体的模型都会把 FFN 的隐藏层维度缩小到原来的 2/3**。  

- 之前我们提到过，传统的 FFN 升维倍数通常是模型维度的 4 倍。 （其实后面的超参数才提到） 
- 换成 GLU 后，维度倍数通常设为 4 的 2/3，也就是 8/3（约 2.67 倍）。比如 LLaMA 70B 设为 2.68 倍，Qwen 14B 设为 2.67 倍。  

为什么要换？

讲义里引用了 Shazeer (2020) 等人的多项消融实验数据： 在保持参数量（223M）和理论算力严格一致的前提下，**SwiGLU 和 GeGLU 在早期 Loss、最终 Loss、以及各种 NLP 验证集（如 SGLUE, XSum 等）上的表现，都极其稳定地优于传统的 ReLU 和 GeLU**。  



虽然不用 GLU 也能训练出好模型（比如强大的 GPT-3 就坚持用了 GeLU），但大量实验表明，换上 SwiGLU 几乎是一笔“稳赚不赔”的买卖，能带来一致的性能增益。这就解释了为什么今天 LLaMA 等架构纷纷将其作为标配。  

### 位置编码

位置编码这个东西的来源其实是，很久之前我们都是RNN，LSTM，当时自然就有这个位置的信息，因为序列是一个一个过来的，但到了attention里面，我们把所有的序列都拼在了一起成了一个大的矩阵，在计算qkv的时候需要知道位置信息，所以就需要直接在这个向量里面加上位置信息就行

关于位置编码有很多，比如绝对位置，相对位置，不过现在RoPE这个very的好用，所以就都用RoPE了，之前的也可介绍一下

![image-20260504234506748](https://cdn.jsdelivr.net/gh/ShiangHou/markdown_image/img/image-20260504234506748.png)

第一个就是绝对位置编码，看起来很简单，在原来的上面直接加就行

在最开始的词向量（Input Embedding）上，**直接加上**一组基于正弦和余弦函数生成的绝对位置向量。

但是算一算就知道，当我们计算attention的时候，会直接出现一些噪音项

讲义指出，当我们计算两个词（位于 $i$ 和 $j$）的内积 $\langle Embed(x, i), Embed(y, j) \rangle$ 时，会产生很多复杂的交叉项（Cross-terms）。这导致模型很难直接感知到纯粹的“相对距离”（$i-j$）

后来又有一些可以学习的绝对位置编码，放弃正弦公式，直接让模型去学习每个绝对位置（0, 1, 2...）对应的向量，然后加到词向量上，但问题是一样的，无法直接获得相对位置



后来一些人提到了相对位置编码

![image-20260504234809878](https://cdn.jsdelivr.net/gh/ShiangHou/markdown_image/img/image-20260504234809878.png)

这种做法是不在最开始加位置信息，而是在计算注意力分数时，强行把一个代表“相对距离”的偏置向量加到计算公式里

但是这个也不好，虽然强迫模型关注相对距离，但破坏了标准内积的形式（如讲义中所说：它不是一个纯粹的内积），计算上也比较麻烦

#### RoPE

重点还是讲RoPE把，首先需要知道什么样的是好的位置编码，首先最好是相对的，然后加上去后进行attention计算后没有杂项

所以我们的目标是，我们希望找到一种位置编码 $f(x, i)$，使得两个词计算注意力（也就是求内积）时，它们的**内积结果只和它们的相对距离 $(i-j)$ 有关**：

$$\langle f(x,i), f(y,j) \rangle = g(x,y,i-j)$$

苏剑林的想法是：向量之间的内积（点乘），本质上是由它们之间的夹角决定的。如果你把两个向量在平面上旋转相同的角度，它们之间的夹角不会变，内积也不会变。

具体来说，RoPE 是这样做的：

1. **分组**：把每个词的 Query 和 Key 向量（比如 128 维），两两一组，切分成 64 个**二维切片**（对应讲义中的 $d=2$）。  
2. **旋转**：把这些二维切片放到复平面（或笛卡尔坐标系）上。对于处在第 $m$ 个位置的词，我们就把它的特征向量旋转 $m \times \theta$ 的角度。  
   - 比如第一个词 "we"，位于位置 0，旋转 $0 \times \theta$。  
   - 第二个词 "know"，位于位置 1，旋转 $1 \times \theta$。  
   - 第三个词 "that"，位于位置 2，旋转 $2 \times \theta$。
3. **相对距离自然涌现**：当位于位置 $m$ 的 Query 和位于位置 $n$ 的 Key 进行点乘计算注意力时，因为它们分别旋转了 $m\theta$ 和 $n\theta$，它们在空间中的**相对夹角就自然地变成了 $(m-n)\theta$**。

这样，RoPE 就在“绝对位置”（我在第几个词）的操作层面上，完美实现了“相对位置”（我们隔了几个词）的内积结果。

假设有三个词，三个token，we know that,然后每个token是128维度

**第一步：切片配对 (Grouping)**

拿 `we` 这个词的向量举例。它有 128 个数字（维度）：$[x_0, x_1, x_2, x_3, ..., x_{126}, x_{127}]$。

RoPE 首先做的是“两两配对”，把它们组合成 64 个二维坐标：

- 第 1 对：$(x_0, x_1)$
- 第 2 对：$(x_2, x_3)$
- ...
- 第 64 对：$(x_{126}, x_{127})$

**第二步：分配不同的“基础旋转速度” ($\theta$)**

这是 RoPE 最核心的设计之一。这 64 对维度，虽然都要旋转，但**它们的“转速”是不一样的**。 RoPE 为每一对维度分配了一个固定的基础角度 $\theta_i$（讲义中那个不断缩小的 $\theta$ 序列）：  

- **最前面的维度（如 0, 1）**：基础角度 $\theta_1$ 非常大（转得最快，比如转一格是 45°）。
- **中间的维度**：基础角度变小（转得中等）。
- **最后面的维度（如 126, 127）**：基础角度 $\theta_{64}$ 极其微小（转得极慢，甚至像没动一样，比如转一格只有 0.1°）。

**第三步：应用位置 ($m$) 进行最终旋转**

现在，我们要根据 token 在句子里的位置 $m$，来决定它们各自转多少度。

最终每个切片旋转的角度公式是：**实际旋转角度 = 位置 $m \times$ 基础角度 $\theta_i$**。

我们来看看这三个词分别发生了什么：

1. **第一个词 `we`（位置 $m=0$）**
   - 对于它的任何维度对，旋转角度都是 $0 \times \theta_i = 0^\circ$。
   - **结论**：`we` 这个词的向量其实**完全没有发生任何旋转**，它保持原样躺在坐标系里。
2. **第二个词 `know`（位置 $m=1$）**
   - 第 1 对 $(x_0, x_1)$：旋转了 $1 \times \theta_1$（比如 45°）。
   - 第 2 对 $(x_2, x_3)$：旋转了 $1 \times \theta_2$（比如 15°）。
   - 第 64 对 $(x_{126}, x_{127})$：旋转了 $1 \times \theta_{64}$（比如 0.1°）。
   - **结论**：`know` 的每个维度切片都转动了“1个单位”的步伐，但前排维度迈的步子大，后排维度迈的步子小。
3. **第三个词 `that`（位置 $m=2$）**
   - 第 1 对 $(x_0, x_1)$：旋转了 $2 \times \theta_1$（比如 90°）。
   - 第 2 对 $(x_2, x_3)$：旋转了 $2 \times \theta_2$（比如 30°）。
   - **结论**：`that` 的每个切片转动了“2个单位”的步伐。

刚才看到的那个大的矩阵，就是所有的两个两个维度之间进行了旋转，m代表的是"we"这个词在sequence的位置，而$\theta$则是在这个自己的token的128维度中的旋转角度

也就是这个升维升4倍，

但是如果我们用了门控的话，GLU激活，我们需要用$\frac{8}{3}$就可以了，因为GLU也会占用一部分

这个就是说多头注意力切分成多少头

$Head\_dim \times Num\_heads = Model\_dim$

这个没啥说的，爱切多少都可以，注意数量要和model dim匹配就行，不要搞什么不一昂样的，或者这个比率在1附近都可以

#### 更宽还是更深

这个深就是说有多少个decoder连环嵌套，然后宽就是说我们的d model是多少

也就是这个比率$d_{model} / n_{layer}$

一般来说，这个值是100-200之间就可以

纯理论上，很多不同深宽比的架构在性能上表现差不多。但是从系统工程（System Concerns）角度看，极其深的模型极难在多个 GPU 之间进行流水线并行（Pipeline Parallelism），并且会导致更高的推理延迟（因为每一层都必须等待上一层算完）。所以，大模型通常会“横向发胖”（增加宽度）以利于并行计算。

#### 词表的大小

这个词表，单语言的肯定是要更小30k - 50k，多语言的肯定要多100k - 250k

#### 正则化

正则化一般都是drop out 或者Weight Decay

drop out的意思就是，直接断掉，即随机把一部分的输出变成0

为什么不用

**数据量太大，根本记不住**：大模型预训练的数据量是数以万亿计的 token，模型在整个训练周期里，几乎看不到重复的数据。既然死记硬背（过拟合）的风险极低，就不需要 Dropout 来防范了。

**拖慢速度，浪费算力**：Dropout 在计算时需要生成大量的随机数掩码（Mask），这不仅消耗额外的计算时间，还浪费了宝贵的显存带宽。在追求极致效率的大模型炼丹炉里，这是不可接受的。



Weight Decay 在数学上通常等价于 **L2 正则化**。它的核心逻辑是：在计算模型的总损失（Loss）时，不仅要看预测得准不准，还要把**模型所有的权重（参数）平方和**作为一个“惩罚项”加进去。

公式大致是：$\text{新 Loss} = \text{预测 Loss} + \frac{\lambda}{2} \sum (\text{权重}^2)$

在梯度下降更新参数时，它表现为：每次更新，不仅要沿着梯度的方向走，还要**强制把当前的权重乘以一个略小于 1 的系数（比如 0.999），让它往 0 的方向缩一点点**。

也就是说直接把权重减小，



为什么LLM要用

**优化动态学（Optimization Dynamics）**：大模型使用特殊的学习率衰减策略（如余弦退火 Cosine schedule）。如果不加 Weight Decay，参数的绝对值会随着训练时间无限膨胀。

**维持稳定，防止崩溃**：当参数极度膨胀时，神经网络的“损失地形（Loss landscape）”会变得极其崎岖陡峭。一点点数据的扰动，就会产生巨大的梯度，导致 Loss 瞬间爆炸（这就是为什么经常看到大模型训练几个月突然崩溃出现 NaN）。

## Stability trick

这个稳定性就是说，数字爆炸，参数爆炸，loss爆炸等，

一般来说，爆炸的来源是那个softmax，因为会有exp，所以很容易炸掉

想一下哪里有softmax!,首先是模型最后会有一个softmax，然后在attention里面有一个

#### Z-loss (Z-损失)

- **问题在哪？** 在模型最后输出词表概率时，我们会用一次 Softmax。Softmax 的分母 $Z$ 是所有词汇得分指数的总和（$Z = \sum e^{U_{r'}(x)}$）。如果模型对某个词极度自信，输出的 Logit 极大，这个 $Z$ 也会变得极大。  
- **怎么解决？** PaLM 模型首创了 `z-loss` 技巧。我们在正常的交叉熵损失之外，额外加上一个极小的惩罚项（辅助损失）：$z\_loss = 10^{-4} \cdot \log^2 Z$。  
- **为什么有效？** 这个正则化项在“逼迫”模型：你可以自信，但你输出的未归一化得分（Logits）不要无脑变大。它鼓励 Softmax 的归一化分母的对数 $\log(Z)$ 尽可能接近于 0（也就是让 $Z$ 接近 1）。这极大地锁定了数值的漂移，提升了训练的稳定性。  
- **谁在使用？** 讲义提到 PaLM、Baichuan 2、DCLM、OLMo 2 等模型都在用。  

####  QK Norm 

- **问题在哪？** 在多头注意力机制中，我们要算 $Q$ 和 $K^T$ 的内积。如果 $Q$ 和 $K$ 的向量范数很大，乘出来的注意力得分在送入 Softmax 前就会大得离谱。
- **怎么解决？** 讲义用了一张搞笑的 meme 图：“栈上更多的 Layer Norms (Stack more Layer Norms)”！它的做法是，在 $Q$ 和 $K$ 算完线性投影后、进行点乘**之前**，额外给 $Q$ 和 $K$ 各做一次归一化（通常是 RMSNorm）。  
- **为什么有效？** 强行把用来计算内积的向量压回正态分布附近，从源头上防止了内积结果过大，从而保护了接下来的 Attention Softmax 不会爆炸。
- **谁在使用？** 这个技巧最早来源于视觉和多模态大模型（如 ViT, Chameleon），现在被引入纯文本模型，如 DCLM、OLMo 2 和 Gemma 2。

#### Logit Soft-capping (Logit 软截断)

- **问题在哪？** 即使有上述技巧，有时候极端数据还是会让 Logit（送入 Softmax 前的得分）冲得很高。
- **怎么解决？** Gemma 2 等模型引入了一种叫 Logit soft-capping 的简单粗暴但极其管用的方法。它利用 $\tanh$ 函数把数值“挤压”到一个硬性的边界内。  
- **公式**：$logits \leftarrow soft\_cap * \tanh(logits / soft\_cap)$。  
- **为什么有效？** 因为 $\tanh$ 的输出永远在 $-1$ 到 $1$ 之间。如果你设置 `soft_cap = 50`，那么无论模型算出来的 Logit 有多大（哪怕是一万），经过这个公式后，它绝对不会超过 50（也不会低于 -50）。这相当于给模型带上了一个绝对安全的数值紧箍咒。  
- **谁在使用？** Gemma 2 在自注意力层设置 `soft_cap = 50.0`，在最后的输出层设置 `soft_cap = 30.0`。虽然讲义也提到这有可能会带来极微小的性能折损（PPL 表格里的轻微浮动），但换来的训练稳定性是无价的。

为了减少要搬运的数据量，业界对传统的 **多头注意力 (MHA, Multi-Head Attention)** 动了刀子。

- **激进派 - MQA (Multi-Query Attention)**：
  - **做法**：不管你有多少个 Query 头（比如 32 个），我**只保留 1 个 Key 头和 1 个 Value 头**！所有的 Query 头都在这一个 KV 头里查信息。  
  - **结果**：KV Cache 的体积瞬间缩小为原来的 1/32，显存读写速度起飞，推理极速变快。  
  - **代价**：讲义中指出，MQA 会带来**轻微的性能下降（PPL 升高）**，因为模型的表达能力被过度压缩了。  
- **折中派（现在的绝对主流）- GQA (Grouped-Query Attention)**：
  - **做法**：不要那么极端。把 Query 头分成几个组，**每个组共享一个 KV 头**。比如 32 个 Q 头，配 8 个 KV 头（4个Q头共用1组KV）。  
  - **结果**：这是一个完美的“甜点”。讲义数据表明，**GQA 的推理速度几乎和极限的 MQA 一样快，但它的模型性能几乎无损，与最原始的 MHA 打成平手**！  
  - **代表模型**：LLaMA 2/3、Mistral、Qwen 绝大部分现代模型全部采用 GQA 架构。

**滑动窗口注意力 (Sliding Window Attention)**

除了在“头”的数量上做文章，讲义还提到了在“看多远”上做限制。  

- **问题**：注意力机制的计算量和上下文长度是平方关系（$O(N^2)$）。上下文如果有 10 万字，计算量会爆炸。
- **做法**：不让模型关注前面的所有词，而是**只关注最近的几个词（比如一个大小为 4096 的滑动窗口）**。就像一个人只看眼前的一页书，不翻前面的。  
- **如何弥补长程记忆？** 因为网络是多层的，底层看了最近 4000 个词，上一层再看最近 4000 个词的聚合特征，通过多层叠加，**感受野（Effective Context Length）会被无限放大**，依然能捕捉长程信息。
