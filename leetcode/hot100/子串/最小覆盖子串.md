## 初步思路

先说题目，题目给了我们两个字符串，一个s一个t，要我们输出s的子串（字串不是子序列，子串需要连续），使得这个子串能够包含t，并且长度是最小的

这里想到之前我们其实可以用，先用hash来取存一下t的，然后存一下s的，

嘶，思路是不是可以和滑动窗口的最大值是一样的，

或者说是KPM算法，鉴定两个字符串是不是匹配的

kmp算法算的是连续匹配的子串吧好像，这里只要求包含即可，顺序也不一样

先回想一下思路，首先我们准备一个hash用来存t，然后准备l和r用来滑动

滑动到一个地方，如果元素在t里面（查找是O1，），那么就加入现在的hash，统计一下

不断的让r向前滑动，直到完全等于t，此时可以弹出，l，如果l也在hash里面，就操作，

重新回顾了一下题目，说是包含t中的每一个

```python 
class Solution:
    def minWindow(self, s: str, t: str) -> str:
        t_dict = {}#准备好存t的
        for i in t:
            if i not in t_dict:
                t_dict[i] = 1
            else:
                t_dict[i] +=1
        s_dict = {}#准备好存s的
        ans = len(s)#最后的答案，最长不过也是s的长度
        l = 0
        for r in range(len(s)):
            #r不断向前吞噬
            if s[r] in t_dict:#只有在t里面的才会被加入‘
                #构建新的s_dict
                if s[r] not in s_dict:
                    s_dict[s[r]] = 1
                else:
                    s_dict[s[r]] += 1
            if s_dict == t_dict:#如果发现相等了，说明找到了一个
                #先统计一下长度
                ans = min(ans,r-l+1)
                #然后让l滚蛋
                while s[l] not in t_dict:#如果l位置是无关的，直接移除
                    l+=1
                #只到移动到l位置是在s_dict里面的，那么去除一个，然后返回
                s_dict[s[l]] -=1
                l+=1
        return ans 
```

有几个问题，首先对于"ADOBECODEBANC"

包含重复的其实也算，就比如BECODEBA，这段里面有两个B，其实也是包含了ABC

我们需要一个新的变量vaild，这个是判断有多少种字符已经满足了t的要求

比如要买三个苹果，我买了4个其实也算对，

所以当我们遍历到一个，它是过的话，vaild就应该+1

直到所有的元素种类和dict的长度是一样的话，就可以



```python 
class Solution:
    def minWindow(self, s: str, t: str) -> str:
        t_dict = {}#准备好存t的
        for i in t:
            if i not in t_dict:
                t_dict[i] = 1
            else:
                t_dict[i] +=1
        s_dict = {}#准备好存s的
        ans = len(s)+1#最后的答案，最长不过也是s的长度
        start = 0#只需要知道长度+开始位置就行了
        vaild = 0#这个是哦安段有多少种字符是满足的
        l = 0
        for r in range(len(s)):
            #r不断向前吞噬
            if s[r] in t_dict:#只有在t里面的才会被加入‘
                #构建新的s_dict
                if s[r] not in s_dict:
                    s_dict[s[r]] = 1
                else:
                    s_dict[s[r]] += 1
            #如果吃进来，发现这个元素已经满足了，就让vaild+1
                if s_dict[s[r]] == t_dict[s[r]]:
                    vaild +=1
                
            #如果vaild满分了，就让l走
            while vaild == len(t_dict):
                #统计长度
                if r-l+1 <ans:
                    ans = r-l+1
                    start = l
                #如果要踢走的是核心字符
                if s[l] in t_dict:
                    #如果踢的时候正好是及格分,即原来s和t都正好是2个，踢完后s就剩一个了，就不合格了
                    if s_dict[s[l]] == t_dict[s[l]]:
                        vaild -=1
                    s_dict[s[l]] -=1
                
                l+=1

        return "" if ans == len(s)+1 else s[start:start+ans]

```
