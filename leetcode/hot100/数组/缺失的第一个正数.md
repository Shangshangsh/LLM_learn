# 缺失的第一个正数

力扣41

## 初步思路

题目说，给我们一个没有排序的数组，nums，让我们找到没有出现的最小的正整数

比如说 1 2 0

那么没有出现的就是3

3 4 -1 1

这里就是 -1 0  1 2 3 4是，然后发现2和0都没有出现，那么就是2，因为要求是正整数

要求用On复杂度和常数的额外空间

这里首先肯定不能排序，因为排序的复杂度是nlogn

可以用一个hash来去做

首先先全部放入hash中，然后遍历hash，对于每一个hash值，找一下i+1和i-1在不在里面，如果都在的话就行，如果有一个不在，列出来，然后找到所有不在的中是正整数且最小的就行

```python 
class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        cache = set()#用一个这个来去存储
        ans = float('inf')
        for num in nums:
            cache.add(num)#先把大家都放在集合里
        
        for i in cache:#然后遍历集合
            if i-1 not in cache and i-1 >0:#找到一个候选的i-1,不在且为正整数
                ans = min(ans,i-1)
            if i+1 not in cache and i+1 >0:#i+1也是，不能用elif，因为都要判断
                ans = min(ans,i-1)
            
        return ans
```

炸裂，理解错题目了，这里是要求不能用额外的空间，此外，比如 5 6 7 8，那么其实最小的应该是1才对，而不是6


## 正确思路

注意到，对于一个数组，长度是N，假设是4，

那么最好的情况就是1 2 3 4，此时缺失的就是5

那如果是 -1 1 3 4 呢，

换句话说，缺失的正整数必然是1到N-1之间，

那么既然如此，我们只需要让符合条件的数都排排队，想1 2 3 4各自站在各自的位置上就行，一旦出现了位置上不是本身的，那么说明这个位置缺人了，直接返回位置下标+1就行

流程是

首先遍历，将当前nums[i]安排在nums[i]-1的位置，比如1 2 3 4，遇到1的话就让他在0位置呆着（当然如果超出了就不用动）

安排完后，比如 -1 1 3 4就会变成。1 -1 3 4

然后继续遍历，顺序遍历，如果发现当前数字和下标对不上，就说名缺人了，直接返回就行

如果没发现，那说明都对了，直接返回n+1就行


```python 
class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        n = len(nums)
        #让各自都放在各自的位置上
        for i in range(n):
            if 1<=nums[i]<=n:#符合范围的才能放
                nums[nums[i]-1],nums[i] = nums[i],nums[nums[i]-1]#交换，吧nums[i]安排在nums[i]-1，比如1安排在nums[0]位置
        
        #然后在遍历一遍
        for i in range(n):
            #i位置上的数应该是i+1，
            if nums[i] != i+1:
                return i
        #如果之前的没返回，说明齐了，那么最小的是n
        return n+1
```
有一个bug
比如[3,4,-1,1]

先换了

-1 4 3 1

再换了 -1 1 3 4

注意到，然后其实是换到了3位置，那个1没换了

所以其实要一直换才行,

        n = len(nums)
