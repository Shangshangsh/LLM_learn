# 前缀树

前缀树是干嘛的

前缀树（Trie，通常读作 "try"，取自 Retrieval）本质上是一种专门为了“高效处理大量字符串”而生的数据结构。

如果我们要存一堆单词（或者词表），最直观的做法是塞进一个数组或者哈希表（Hash Set/Map）。哈希表查“精准匹配”极快（比如查词典里有没有 "apple"），但它有一个致命弱点：无法处理“前缀匹配”。如果我想问“词典里有没有以 app 开头的词？”，哈希表就彻底瞎了，只能把几百万个词全都遍历一遍。


核心思想：共享前缀，白嫖空间
想象一下，你有三个词：cat（猫）、car（车）、camera（相机）。
如果用普通数据结构存，你需要独立存下所有的字母。

但在前缀树里，它们是这样“长”出来的：

大家都以 c 开头，所以从树根牵出一条写着 c 的树枝。

下一个字母都是 a，所以顺着 c 再牵出一条写着 a 的树枝。此时，这段路径 c -> a 是这三个词完全共享的。

到了第三个字母，大家分道扬镳了：从 a 牵出 t 形成 cat，牵出 r 形成 car，牵出 m 继续往后长形成 camera。

发现了吗？长得像的单词，前面的路是合并共用的。 越是前缀相同的词，省下的空间就越多，查找起来也越顺藤摸瓜。



## 实现思路

insert：例如先插入字符串 aab，相当于生成了一条移动方向为「左-左-右」的路径。标记最后一个节点为终止节点。再插入字符串 aabb，相当于生成了一条移动方向为「左-左-右-右」的路径。标记最后一个节点为终止节点。
search：例如查找字符串 aab，相当于查找二叉树中是否存在一条移动方向为「左-左-右」的路径，且最后一个节点是终止节点。
startsWith：例如查找前缀 aa，相当于查找二叉树中是否存在一条移动方向为「左-左」的路径，无其他要求。

所以说，扩展到26个字母，这就是一个26叉树

## 实现代码

首先是初始化，我们可以用一个字典来去实现

```python 
class TrieNode:
    def __init__(self):
        # children 里存的是 { 'a': TrieNode(), 'b': TrieNode(), ... }
        self.children = {} 
        # 标记从根节点走到这里，是否构成了一个完整的插入过的单词
        self.is_end = False

        self.root = TrieNode()#哑巴节点，所有的树都是从这开始的
```

### insert

就是需要吧一个单词给放进去
目标：把单词 apple 放进树里。
动作：从根节点出发，遍历 apple 的每一个字符。
如果当前字符不在子节点字典里？那就为它新建一个节点。然后继续顺着藤往上爬。遍历完所有字符后，在最后一个节点处，把 is_end 设为 True。

```python 
def insert(self,word):
    node = self.root
    #遍历每一个字母
    for char in word:
        if char not in node.childern:
            node.childern[char] = TrieNode()
        node = node.childern[char]#顺着路往下走
    
    #走完后，记录一下,这是一个完整的树
    node.is_end = True

```

### search

目标：查找树里有没有插入过完整的 apple。
动作：和插入一模一样，顺着字母往下找。如果半路发现路断了（比如找 b，但字典里没这条路），直接返回 False。
关键点：如果顺利走到了最后，能不能直接返回 True？不能！ 比如树里只有 apple，你现在搜索 app，你确实能顺着路径走完，但你停下的那个节点，并没有挂着单词结尾的牌子（is_end == False）。所以最后必须返回当前节点的 is_end 状态。

```python 
def search(self,word):
    node = self.root
    for char in word:
        if char not in node.childern:
            return False
        node = node.childern
    return node.is_end
```


### srartwith

目标：树里有没有以 app 开头的单词？
动作：逻辑和 Search 几乎完全一样！唯一的区别是，只要你能顺畅地把前缀走完，中间没断路，这就说明肯定有以它开头的单词（不管是 app 本身，还是后面的 apple），直接返回 True 即可，不需要管 is_end 牌子。

```python 
def startwith(self,pre):
    node = self.root
    for char in pre:
        if char not in node.childern:
            return False
        node = node.childern
    return True

```

```python 
class Trie:
    def __init__(self):
        self.is_end = False
    def insert(self, word: str) -> None:
        node = self.root
        for char in word:
            if char not in node.childern:
                node.childern[char] = TrieNode()
            node = node.childern
        node.is_end = True

    def search(self, word: str) -> bool:
        node = self.root
        for char in word:
            if char not in node.childern:
                return False
            node = node.childern
        return node.is_end
    def startsWith(self, prefix: str) -> bool:
        node = self.root
        for char in prefix:
            if char not in node.childern:
                return False
            node = node.childern
        return True
```
