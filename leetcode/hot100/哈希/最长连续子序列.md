那我先遍历一遍存在hash中，然后再遍历一遍，这样子就可以了呀，这样就可以解决刚才的问题了，

思路就是，我依旧是先存储一个used用来（也可以不存储，无所谓感觉）

遍历一个，如果+1在hash，那么就继续遍历那个在的+1看看在不在，这个过程维护一共cur_max，直到不在了，然后继续取max，往下走就行，不过很容易搞成n2，所以还是used吧，存下标，下标用过的就不用了

```python 
class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        n = len(nums)
        a = {}
        for i in range(n):
            if nums[i] not in a:
                a[nums[i]] = i#有重复的下标就刷过去无所谓吧，这里它有重复的也不统计
        used = [0]*n
        ans = 0
        for i in range(n):
            if used[i] ==1:#如果这个用过了
                continue
            else:#没用过
                cur_max = 1
                maybe_next = nums[i]+1#先把可能的下标存储了
                while maybe_next in a:
                    used[a[maybe_next]] = 1#记录用过的
                    cur_max+=1
                    maybe_next+=1
                ans = max(ans,cur_max)
        return ans
```

```python 
class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        a = set(nums)#直接set起手就行
        n = len(a)#这里n也可也换一下，去重
        ans = 0
        for num in a:
            if num-1 in a:#如果这个不是一个所谓的起点
                continue
            else:
                cur_max = 1
                maybe_next = num+1
                while maybe_next in a:
                    cur_max+=1
                    maybe_next+=1
                    
                ans = max(ans,cur_max)
        return ans

```
